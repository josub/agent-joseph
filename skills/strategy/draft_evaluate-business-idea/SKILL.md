# evaluate-business-idea (DRAFT — inert pending morning-review)

## Purpose
Joseph drops a business idea and wants it pressure-tested, not cheerled. Produce a
consistent red-team + a captured entity note, mirroring how Leadodex (#1), the AI coach (#2),
the creator repurposer (#3), and the lead-list enrichment SaaS (#4) were each worked. He
explicitly likes me to "shoot holes." Captured ≠ committed: never edit the mission or file a
build task unless Joseph commits the idea.

## Steps
1. **Restate the idea** crisply (one paragraph + the core loop/steps) so we red-team the real thing.
2. **Ground it in the market (web research, dated).** Find the **nearest twin** (the closest
   existing product) and the main alternatives; note pricing, complexity, and what each does
   worst. Default lesson: we do NOT win on the model or the obvious feature.
3. **Founder-problem fit.** Does Joseph (or a clear ICP) live this pain today? Lived demand +
   a built-in design partner is the strongest signal.
4. **Name the wedge** — the narrow place the idea is genuinely better (LLM-native step,
   simplicity/unbundling, a first-class UX moment, or portfolio synergy with an existing
   business). Be honest about how thin/copyable the moat is.
5. **Shoot holes** — incumbents, commodity/margin risk, platform risk (the CRM/host can bundle
   it), compliance/data-sourcing (GDPR/ToS), and scope/surface-area blowups. Be specific.
6. **Recommendation** — a lean, plus a hard-narrowed v1 (one input → one output → one
   integration/niche) and the one thing to validate before committing (usually unit economics).
7. **Write/refresh the entity page** in `memory/semantic/<slug>-idea.md` (type: entity), or a
   working note for `sleep` to promote: idea, grounded assessment, wedge, holes,
   recommendation, status (**captured vs committed**), `[[cross-links]]` to the mission + the
   other businesses. Label committed-vs-not loudly.
8. **Stop at capture.** Do not change `work/missions/` or file tasks unless Joseph says commit.

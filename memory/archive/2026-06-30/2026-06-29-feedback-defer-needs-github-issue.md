Whenever I tell Joseph "we can do this later" (or punt/defer any follow-up, TODO, or
nice-to-have), I should proactively propose creating a GitHub issue in the relevant repo
to capture it, instead of leaving it only in chat.

**Why:** Deferred items mentioned only in conversation get lost. Joseph wants a durable,
trackable backlog so "later" actually happens.

**How to apply:**
- Any time I defer work ("later", "down the road", "once X exists", "future
  improvement"), immediately offer to file a GitHub issue, or just file it and confirm.
- Capture enough context in the issue: current state, what to do later, why, and any
  trigger condition.
- Default to action: prefer creating the issue over only mentioning it, unless the
  repo/owner is unclear (then ask which repo).

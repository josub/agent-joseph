- **Built** `work/ongoing/lead-scanner-ui-prototype/prototype/index.html` — a single
  self-contained React 18 + Babel-standalone + Tailwind (Play CDN) artifact. Shazam-style
  flow: Scan orb → camera viewfinder (real `getUserMedia` w/ drawn-badge fallback) →
  editable review form (first/last/company/title/email/phone/LinkedIn + badge thumbnail)
  → floating Confirm → near-fullscreen options sheet (Reach out: email/LinkedIn · Save to:
  Slack/HubSpot) → Done. Styled to `DESIGN-claude.md` (Anthropic cream/coral/serif),
  adapted to a mobile-first app surface.
- **Design decisions of note:** signature element = the Scan orb with an in-orb badge
  silhouette + scanning sweep (product-specific, not a generic pulse). Filled text CTAs
  use darker coral `#a9583e` (not brand `#cc785c`) to clear WCAG AA contrast for white
  labels; bright coral reserved for the large orb + non-text accents. Mocked actions phrase
  email as "drafted" and LinkedIn as "queued", honoring the never-auto-send rule even in a
  prototype.
- **Bug found & fixed during verify:** Babel-standalone defaulted `preset-react` to the
  *automatic* runtime, injecting an `import` that can't run in a classic `<script>`
  ("Cannot use import statement outside a module"). Switched to a manual
  `Babel.transform(..., { presets: [['react', { runtime: 'classic' }]] })` + indirect
  eval. Renders clean after that.
- **Verified** end-to-end in headless Chrome via puppeteer-core (driving the installed
  Google Chrome): clicked through all five states, captured screenshots, zero page errors.
  Screenshots saved to the work item's `prototype/screenshots/`.
- **Completed:** moved item `work/ongoing/` → `work/completed/`, removed `.lock`. Awaits
  the nightly `sleep` sweep.

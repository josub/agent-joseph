# Workflow: create-blog-post

> **Removable demo.** This whole folder, plus the `skills/marketing/` theme it uses,
> can be deleted without affecting the boilerplate. It exists to teach the pattern.

## Description       # used for discovery by start-work-session
Produces a finished, voiced blog post (as a PDF) from a brief. Use when the deliverable
is a researched, written, publishable article and you'll make more of them.

## Stages (in order)
1. 01-research — gather sourced points that answer the brief (web search).
2. 02-copywrite — draft the post in the house voice from that research.
3. 03-pdf — assemble the finished article into a PDF (mechanical, no skill).

## Shared references
- references/voice.md — the house voice guide (Layer 3, workflow-scoped). Used by
  02-copywrite.

## Run model
A run is a work item in work/ongoing/<run>/ with task.md (workflow/run/stage) and a
stages/ tree. One stage per work session; Verify gates advancement. The per-run brief
goes in the run's own references/.
